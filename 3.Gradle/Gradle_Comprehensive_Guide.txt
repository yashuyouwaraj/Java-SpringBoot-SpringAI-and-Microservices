1. WHAT IS GRADLE?
================================================================================

Gradle is a powerful, flexible, and fast build automation tool that's become the
standard for Java, Android, and many other types of projects. It's written in Java
and Groovy, making it highly extensible and customizable.

Key Characteristics:
- Build automation tool
- Written in Java and Groovy
- Uses Groovy-based DSL (Domain Specific Language)
- Supports multiple languages and platforms
- Highly extensible and customizable

================================================================================
2. WHY DO WE NEED GRADLE?
================================================================================

Build Tools Evolution:
1. Manual compilation (javac, jar, etc.) - Error-prone, time-consuming
2. Ant (2000) - XML-based, procedural, no dependency management
3. Maven (2004) - Convention over configuration, XML-based, dependency management
4. Gradle (2007) - Best of both worlds, Groovy DSL, highly flexible

Why Gradle is Needed:
- Automates repetitive build tasks
- Manages dependencies automatically
- Provides consistent build process across teams
- Supports multiple languages and platforms
- Enables continuous integration/deployment
- Faster builds with incremental compilation
- Better IDE integration
- Rich ecosystem of plugins

3. HOW TO CREATE GRADLE PROJECTS
================================================================================

Method 1: Using Gradle Init Command
-----------------------------------
# Create new directory
mkdir my-gradle-project
cd my-gradle-project

# Initialize Gradle project
gradle init

# Choose options:
# - Type: application
# - Language: Java
# - Build script: Groovy
# - Test framework: JUnit Jupiter

Method 2: Manual Project Structure
----------------------------------
Create these files manually:

my-project/
├── build.gradle          # Main build script
├── settings.gradle       # Project settings
├── gradle/
│   └── wrapper/
│       ├── gradle-wrapper.jar
│       └── gradle-wrapper.properties
└── src/
    ├── main/
    │   ├── java/         # Java source files
    │   └── resources/    # Resources
    └── test/
        ├── java/         # Test source files
        └── resources/    # Test resources

Method 3: Using IDE
-------------------
- IntelliJ IDEA: File → New → Project → Gradle
- Eclipse: File → New → Other → Gradle → Gradle Project
- VS Code: Use Gradle extension

4. GRADLE BUILD SCRIPT (build.gradle)
================================================================================

Basic build.gradle Structure:

plugins {
    id 'java'                    // Java plugin
    id 'application'             // Application plugin
}

group = 'com.example'
version = '1.0-SNAPSHOT'
sourceCompatibility = '11'

repositories {
    mavenCentral()               // Dependency repositories
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web:3.2.0'
    testImplementation 'org.junit.jupiter:junit-jupiter:5.10.0'
}

application {
    mainClass = 'com.example.App'  // Main class for application plugin
}

tasks.named('test') {
    useJUnitPlatform()           // Configure test task
}

5. GRADLE COMMANDS
================================================================================

Basic Commands:
---------------
gradle build              # Build the project (compile, test, package)
gradle clean               # Delete build directory
gradle clean build         # Clean and build
gradle test                # Run tests
gradle compileJava         # Compile Java sources
gradle compileTestJava     # Compile test sources

Application Commands:
--------------------
gradle run                 # Run the application (requires application plugin)
gradle bootRun             # Run Spring Boot application
gradle distTar             # Create distribution tar
gradle distZip             # Create distribution zip

Dependency Commands:
-------------------
gradle dependencies        # Show dependency tree
gradle dependencyInsight --dependency spring-core  # Show why a dependency is used

Wrapper Commands:
----------------
gradle wrapper             # Generate Gradle wrapper
./gradlew build            # Use wrapper (Linux/Mac)
gradlew.bat build          # Use wrapper (Windows)

Information Commands:
--------------------
gradle tasks                # List all available tasks
gradle projects             # Show project structure
gradle properties           # Show project properties
gradle --version            # Show Gradle version

Debugging Commands:
------------------
gradle build --info         # Verbose output
gradle build --debug        # Debug output
gradle build --stacktrace   # Show stack traces on errors
gradle build --scan         # Generate build scan

6. GRADLE VS MAVEN COMPARISON
================================================================================

Configuration:
--------------
Gradle: Groovy/Kotlin DSL - More readable, flexible
Maven: XML - Verbose, less flexible

Build Performance:
------------------
Gradle: Incremental builds, build cache, daemon
Maven: Full rebuilds each time

Dependency Management:
---------------------
Gradle: More flexible, supports dynamic versions
Maven: Strict version management

Learning Curve:
---------------
Gradle: Steeper (Groovy knowledge needed)
Maven: Easier (XML is straightforward)

Extensibility:
-------------
Gradle: Highly extensible with custom tasks/plugins
Maven: Limited extensibility

IDE Support:
-----------
Both: Excellent support in all major IDEs

Community & Ecosystem:
---------------------
Maven: Larger community, more plugins
Gradle: Growing rapidly, modern features

When to Use Gradle:
- Large projects with complex builds
- Multi-language projects
- Need for custom build logic
- Performance-critical builds
- Android development (official)

When to Use Maven:
- Simple Java projects
- Team prefers XML configuration
- Existing Maven infrastructure
- Need for strict standardization

7. GRADLE FEATURES
================================================================================

Core Features:
--------------
1. Incremental Builds: Only rebuilds changed parts
2. Build Cache: Reuses outputs from previous builds
3. Daemon: Keeps Gradle process running for faster builds
4. Parallel Execution: Run tasks in parallel
5. Composite Builds: Build multiple projects together

Advanced Features:
-----------------
1. Custom Tasks: Create your own build tasks
2. Plugins: Extend functionality (Java, Spring, Android, etc.)
3. Multi-project Builds: Manage multiple modules
4. Dependency Resolution: Intelligent dependency management
5. Configuration Injection: Runtime configuration
6. Build Scans: Detailed build analysis and debugging

Language Support:
----------------
- Java, Kotlin, Groovy, Scala
- Android, Spring Boot
- C++, JavaScript, Python (via plugins)
- Docker, Kubernetes integration

8. GRADLE WRAPPER
================================================================================

What is Gradle Wrapper?
----------------------
- Ensures consistent Gradle version across team
- No need to install Gradle globally
- Version controlled with project

Creating Wrapper:
----------------
gradle wrapper --gradle-version 8.5

Wrapper Files:
-------------
- gradlew (Unix script)
- gradlew.bat (Windows script)
- gradle/wrapper/gradle-wrapper.jar
- gradle/wrapper/gradle-wrapper.properties

Usage:
------
# Instead of 'gradle build'
./gradlew build    # Linux/Mac
gradlew.bat build  # Windows

10. GRADLE DEPENDENCY MANAGEMENT
================================================================================

Dependency Types:
----------------
- implementation: Compile and runtime (not exposed to consumers)
- api: Compile and runtime (exposed to consumers)
- compileOnly: Compile time only
- runtimeOnly: Runtime only
- testImplementation: Test dependencies

Version Management:
------------------
- Fixed versions: '1.0.0'
- Dynamic versions: '1.+', 'latest.release'
- Version ranges: '[1.0, 2.0)'

Transitive Dependencies:
-----------------------
- Gradle resolves all transitive dependencies
- Handles version conflicts automatically
- Can exclude specific dependencies

11. GRADLE TASKS
================================================================================

Built-in Tasks:
---------------
- build: Full build (compile, test, package)
- clean: Delete build outputs
- check: Run verification tasks
- assemble: Create archives
- test: Run tests

Custom Tasks:
-------------
task hello {
    doLast {
        println 'Hello, Gradle!'
    }
}

Task Dependencies:
-----------------
task taskB {
    dependsOn taskA
    doLast {
        println 'Task B executed'
    }
}

13. TROUBLESHOOTING GRADLE
================================================================================

Common Issues:
--------------
1. Build failures: Check stacktrace with --stacktrace
2. Dependency conflicts: Use dependencyInsight task
3. Slow builds: Enable daemon, check for bottlenecks
4. Memory issues: Increase heap size with GRADLE_OPTS
5. Cache issues: Clean cache with gradle --stop (stops daemon)

Debug Commands:
---------------
gradle build --scan          # Online build scan
gradle build --profile       # Performance profile
gradle dependencies --configuration compileClasspath
